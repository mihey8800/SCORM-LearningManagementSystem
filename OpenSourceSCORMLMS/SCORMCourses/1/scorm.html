Введение в ОМСИС (SCORM)

Объектная модель совместно используемого содержания (ОМСИС) ( The Sharable Content Object Reference Model  - SCORM) служит для создания повторно используемых учебных материавло как "учебных элементов" с общими техническими требованиями к компьютерным учебным программам и онлайн-обучающим сайтам. ОМСИС представляет технические требования как свод руководств, спецификаций и стандартов.

ОМСИС помогает определить технические условия дл разработки онлайн-обучающих сред. Это модель, которая представляет собой набор взаимосвязанных технических спецификаций и руководств, спроектированная для удовлетворения высокоуровневых требований к учебным материалам и системам.  ОМСИС описывает модель объединения содержания ("Content Aggregation Model (CAM)") и средства управления работой программы ("Run-Time Environment (RTE)") для учебных объектов, чтобы поддерживать соответствующее представление учебных материалов, основываясь на критериях целей, предпочтений и произыодительности учащегося.

ОМСИС определяет Web как основную среду для передачи знаний. Все, что размещено в Web может быть легко использовано в других образовательных средах (программах), предъявляющих меньше требований к доступности и использованию сетевых технологий. Благодаря данной стратегии, большая часть работы по переносу на современные технические и программные платформы отпадает, так как Web является универсальной средой. Развивая существующие Web стандарты, ОМСИС позволяет разработчикам уделить больше внимания эффективным образовательным стратегиям.

Разработка и развитие ОМСИС продолжается. На текущий момент ОМСИС предлагает программный интерфейс (Application Programming Interface - API) для представления информации об использовании учебных объектов учащимся.  Этот интерфейс предлагает определнную модель данных для представления такой информации, спецификацию переноса учебных материалов независимо от технической и программной платформы. Предлагает набор метаданных, которые могут быть использованы для описания учебных элементов и набор стандартных последовательных правил, применимых к организации учебных элементов.

Первый выпуск ОМСИС был в январе 2000. Он продолжает обновляться и расширять круг спецификаций, благодаря сотрудничеству с промышленными, правительственными и образовательными организациями.

ОМСИС сегодня
ОМСИС - это набор спецификаций и стандартов. Практически все  спецификации и стандарты представлены разными организациями. Они все сгруппированы в 3 основных категории: модель объединения содержания ("Content Aggregation Model (CAM)"), средства управления работой программы ("Run-Time Environment (RTE)") и последовательность и навигация ("Sequencing and Navigation (SN)) (представлена в ОМСИС 2004).

Материалы в каждой из указаных категорий сфокусированы на определенных аспектах ОМСИС, но есть вопросы, где они пересекаются. Например, категория "Средства управления работой программы" сосредоточены на связи между наполнением и системой управления обучением (Learning Management System).
